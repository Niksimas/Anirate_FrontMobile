# WebSocket API — Anirate

## Подключение

```
WS /api/v1/ws?token={access_token}
```

- Токен передаётся как query-параметр
- Сервер валидирует токен, определяет `user_id`
- Сервер автоматически подписывает пользователя на события его друзей и его списков
- Клиент не отправляет сообщения серверу (только получает)

## Формат сообщений

```json
{
  "type": "event_name",
  "payload": { ... }
}
```

## Правила

1. **Не отправлять событие автору действия.** Если пользователь сам добавил аниме в список — ему WS-событие не нужно, он уже обновил UI локально.
2. **Фильтрация на сервере.** Клиент не подписывается на конкретные списки или пользователей. Сервер сам знает, в каких списках состоит пользователь и кто его друзья, и отправляет только релевантные события.
3. **Множественные подключения.** Один пользователь может иметь несколько активных WebSocket-соединений (например, с разных устройств). Событие отправляется во все его соединения.
4. **Закрытие при невалидном токене.** Если токен протух или невалиден — сервер закрывает соединение с кодом `4401`. Клиент рефрешит токен и переподключается.

## События: Друзья

### `friend:request:created`

Отправляется получателю заявки в друзья.

```json
{
  "type": "friend:request:created",
  "payload": {
    "request_id": 42,
    "sender_id": 5,
    "full_name": "Иван Петров",
    "picture": "https://lh3.googleusercontent.com/a/photo123"
  }
}
```

### `friend:request:accepted`

Отправляется тому, чью заявку приняли.

```json
{
  "type": "friend:request:accepted",
  "payload": {
    "request_id": 42,
    "user_id": 12,
    "full_name": "Маша Сидорова",
    "picture": "https://lh3.googleusercontent.com/a/photo456"
  }
}
```

### `friend:removed`

Отправляется тому, кого удалили из друзей.

```json
{
  "type": "friend:removed",
  "payload": {
    "friend_id": 5
  }
}
```

## События: Общие списки — аниме

### `list:anime:added`

Кто-то добавил аниме в общий список. Отправляется всем участникам списка (кроме автора).

```json
{
  "type": "list:anime:added",
  "payload": {
    "list_id": 3,
    "anime": {
      "anime_id": 228,
      "title": "Магическая битва",
      "image_url": "https://animego.me/media/cache/thumbs_300x420/upload/anime/images/1.jpg",
      "season": "Осень",
      "year": 2020,
      "anime_type": "TV Сериал",
      "added_by": 5,
      "added_at": "2026-03-27T14:30:00",
      "ratings": [],
      "average_score": null
    }
  }
}
```

### `list:anime:removed`

Аниме удалено из списка. Отправляется всем участникам (кроме автора).

```json
{
  "type": "list:anime:removed",
  "payload": {
    "list_id": 3,
    "anime_id": 228
  }
}
```

## События: Общие списки — оценки

### `list:rating:updated`

Кто-то поставил или изменил оценку аниме в списке. Отправляется всем участникам (кроме автора).

```json
{
  "type": "list:rating:updated",
  "payload": {
    "list_id": 3,
    "anime_id": 228,
    "rating": {
      "user_id": 5,
      "full_name": "Иван Петров",
      "picture": "https://lh3.googleusercontent.com/a/photo123",
      "score": 9,
      "comment": "Огонь",
      "updated_at": "2026-03-27T15:00:00"
    },
    "new_average_score": 8.5
  }
}
```

### `list:rating:deleted`

Оценка удалена. Отправляется всем участникам (кроме автора).

```json
{
  "type": "list:rating:deleted",
  "payload": {
    "list_id": 3,
    "anime_id": 228,
    "user_id": 5,
    "new_average_score": 7.2
  }
}
```

## События: Общие списки — участники

### `list:member:joined`

Новый участник вступил в список. Отправляется всем текущим участникам.

```json
{
  "type": "list:member:joined",
  "payload": {
    "list_id": 3,
    "member": {
      "user_id": 12,
      "full_name": "Маша Сидорова",
      "picture": "https://lh3.googleusercontent.com/a/photo456",
      "joined_at": "2026-03-27T16:00:00"
    }
  }
}
```

### `list:member:left`

Участник вышел или был кикнут. Отправляется всем оставшимся участникам. Если участника кикнули — ему тоже отправляется (чтобы он закрыл экран списка).

```json
{
  "type": "list:member:left",
  "payload": {
    "list_id": 3,
    "user_id": 12
  }
}
```

## События: Общие списки — метаданные

### `list:updated`

Название или описание списка изменены. Отправляется всем участникам (кроме автора).

```json
{
  "type": "list:updated",
  "payload": {
    "list_id": 3,
    "name": "Новое название",
    "description": "Новое описание"
  }
}
```

### `list:deleted`

Список удалён владельцем. Отправляется всем участникам (кроме владельца).

```json
{
  "type": "list:deleted",
  "payload": {
    "list_id": 3
  }
}
```

---

## Коды закрытия соединения

| Код | Значение | Действие клиента |
|-----|----------|------------------|
| 1000 | Нормальное закрытие | Переподключиться |
| 1001 | Сервер перезагружается | Переподключиться с задержкой |
| 4401 | Токен невалиден/протух | Рефрешить токен, переподключиться |
| 4403 | Доступ запрещён | Не переподключаться |
